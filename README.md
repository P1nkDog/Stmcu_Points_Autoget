# STMCU 自动积点脚本

## 目的

本脚本旨在自动化完成 STMCU 官网 `https://www.stmcu.com.cn` 的视频观看任务，帮助用户每日自动获取积点，节省手动操作时间。

## 功能

### 核心功能

- **自动浏览视频**：按顺序访问视频页面（video/1, video/2, ...）
- **积点检测**：精确定位 `<font class="zt_r">` 元素获取积点变化
- **进度保存**：断点续传，下次运行自动从上次位置继续
- **日志记录**：详细记录每个视频的观看状态和积点变化
- **智能停止**：达到每日积点上限（300点）自动停止

### 辅助功能

- **多浏览器支持**：支持 Edge 和 Chrome 浏览器
- **多用户支持**：每个用户独立的进度文件和日志
- **自动获取今日积点**：从积点详情页自动统计今日已获积点
- **手动登录**：支持用户在浏览器中手动登录，保持会话
- **错误处理**：遇到验证码或网络错误时记录日志并通知用户
- **状态标记**：区分已看视频（获得积点）和无效视频（未获得积点）
- **反爬检测**：随机 User-Agent、随机延迟、隐藏 WebDriver 标志
- **智能等待**：使用 WebDriverWait 替代固定等待，适应不同网络环境
- **配置化支持**：通过配置文件和命令行参数自定义各项设置

## 使用说明

### 环境要求

- **操作系统**：Windows 10/11
- **Python**：3.8 或更高版本
- **浏览器**：Microsoft Edge 或 Google Chrome（任选其一）

### 第一步：安装 Python 依赖

打开终端（CMD 或 PowerShell），进入项目目录，执行以下命令：

```bash
pip install -r requirements.txt
```

如果安装速度较慢，可以使用国内镜像源：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 第二步：运行脚本

```bash
python main.py
```

**首次运行**时，脚本会提示选择浏览器：

```
==================================================
首次运行配置
==================================================

请选择要使用的浏览器:
  1. Edge (推荐)
  2. Chrome

请输入选项 (1 或 2): 
```

选择后，浏览器选择会保存到 `config.json`，下次运行无需重新选择。

### 第三步：登录 STMCU 账号

脚本会自动打开 STMCU 登录页面，请在浏览器中手动登录：

1. 在浏览器中输入账号密码
2. 完成登录
3. 回到终端，按 **回车键** 继续

### 第四步：等待脚本执行

登录后，脚本会自动执行以下操作：

1. **获取用户名**：从网页自动识别当前登录用户
2. **加载用户进度**：读取该用户的独立进度文件
3. **获取今日积点**：自动统计今日已获得的积点数
4. **检查积点上限**：如果已达到 300 点，会提醒并退出
5. **开始视频任务**：自动循环观看视频获取积点

### 第五步：查看结果

脚本执行完成后，会显示今日任务总结：

```
==================================================
任务完成总结
==================================================
用户: P1nkDog
今日获取积点: 120
处理视频数量: 15
有效视频数量: 12
下次起始视频ID: 16
==================================================
```

日志文件保存在 `logs_{用户名}/` 目录下，格式为 Markdown。

### 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--max-points` | 每日积点上限 | 300 |
| `--points-per-video` | 每个视频获得的积点数 | 10 |
| `--delay-min` | 最小延迟秒数 | 0.5 |
| `--delay-max` | 最大延迟秒数 | 2.0 |
| `--timeout` | 页面加载超时秒数 | 10 |
| `--debug` | 启用调试模式 | False |
| `--browser` | 选择浏览器 (chrome/edge) | 配置文件 |
| `--reset-browser` | 重新选择浏览器 | False |
| `--retries` | 视频404时的刷新重试次数 | 3 |

### 常用命令示例

```bash
# 使用默认配置运行
python main.py

# 使用 Chrome 浏览器
python main.py --browser chrome

# 重新选择浏览器
python main.py --reset-browser

# 设置每日上限 200 积点
python main.py --max-points 200

# 启用调试模式（查看详细执行信息）
python main.py --debug

# 调整延迟速度（更快）
python main.py --delay-min 0.3 --delay-max 1.5

# 调整延迟速度（更安全）
python main.py --delay-min 1 --delay-max 3

# 组合使用
python main.py --browser chrome --max-points 200 --debug
```

### 多用户支持

脚本支持多用户使用，每个用户拥有独立的文件：

- `progress_{用户名}.json` - 用户进度文件
- `logs_{用户名}/` - 用户日志目录

脚本会自动从网页获取用户名，无需手动输入。

---

### 积点上限检查

每次启动时，脚本会自动检查用户今日已获积点：

- 如果已达到 300 点上限，会显示警告并退出
- 提醒用户继续观看视频不会增加积点
- 每个视频只有第一次观看能获得积点

---

## 积点规则

| 任务类型 | 积点 |
|---------|------|
| 观看视频 | +10 |
| 下载视频 | +20 |
| 活动/培训报名 | +5 |
| 活动/培训签到 | +15 |
| 峰会/研讨会签到 | +30 |
| 分享 | +50（需管理员确认）|
| 每日签到 | +2 |

**每日上限**：300 积点

## 实现逻辑

### 1. 初始化阶段

```
启动程序
    │
    ├── 加载配置文件 (config.json)
    │   └── 首次运行时选择浏览器
    │
    ├── 解析命令行参数 (argparse)
    │
    ├── 初始化浏览器
    │   ├── Edge: 使用 webdriver-manager 自动下载
    │   └── Chrome: 使用 webdriver-manager 自动下载
    │
    └── 等待用户登录
```

### 2. 用户识别阶段

```
登录完成后
    │
    ├── 从网页获取用户名
    │   └── 查找 .common-h-login 中的用户名
    │
    ├── 加载用户独立进度
    │   └── progress_{username}.json
    │
    └── 自动获取今日积点
        ├── 访问积点详情页
        ├── 解析表格统计今日积点
        └── 检查是否已达上限
```

### 3. 积点检测阶段

```
访问积点页面 (/User/MyPoint)
    │
    ├── 等待页面加载 (WebDriverWait)
    │
    ├── 多策略精确定位
    │   ├── 策略1: 查找 <font class="zt_r"> 元素
    │   ├── 策略2: 查找 div.lb1b 中的"可用积点"
    │   ├── 策略3: 全文正则匹配"可用积点\s*(\d+)"
    │   └── 策略4: 调试模式输出页面结构
    │
    └── 返回积点值（失败则手动输入）
```

### 4. 视频任务循环

```
开始循环（当今日积点 < 上限）
    │
    ├── 记录访问前积点
    │
    ├── 访问视频页面 (/video/{id})
    │   ├── 使用 WebDriverWait 等待页面加载
    │   ├── 检查页面是否有效（非404）
    │   └── 检查是否有视频内容
    │
    ├── 随机等待（积点更新延迟）
    │
    ├── 获取访问后积点
    │
    ├── 计算积点变化
    │   ├── points_gained > 0 → 标记为"已看"
    │   └── points_gained = 0 → 标记为"无效"
    │
    ├── 记录日志
    │
    ├── 保存用户进度
    │
    └── 随机延迟（模拟人类行为）
```

### 5. 结束阶段

```
循环结束
    │
    ├── 保存日志到 logs_{username}/log_日期.md
    ├── 保存最终进度
    ├── 关闭浏览器
    │
    └── 显示总结
        ├── 用户名
        ├── 今日获取积点
        ├── 处理视频数量
        ├── 有效视频数量
        └── 下次起始视频ID
```

## 注意事项

1. **登录状态**：请确保在脚本运行期间不要关闭浏览器或退出登录
2. **验证码**：如果遇到验证码，脚本会暂停并提示用户处理
3. **网络问题**：遇到网络或404错误时会自动重试，重试耗尽后标记为无效并继续下一个视频
4. **积点上限**：每日最多获取 300 积点，超过自动停止
5. **进度保存**：脚本会自动保存用户进度，下次运行自动继续
6. **反爬措施**：脚本已内置随机延迟和反检测机制
7. **调试模式**：遇到问题时可使用 `--debug` 参数查看详细信息
